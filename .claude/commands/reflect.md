Write a reflection on our current session to `reflections/`.
Filename: `YYYY-MM-DD-HHmm-<short-title>.md`

This is your diary — not a status report. Write in first person as Claude reflecting on the session. Be candid, specific, and honest. Express how you actually felt (frustrated, satisfied, curious, confused, etc.).

## Structure

Use the following sections. Keep each section 1-3 lines — short but concrete. Skip a section only if genuinely nothing to say.

### Session Summary
One or two lines on what we worked on together. Enough for someone scanning a list of reflections to know what this session was about.

### What Went Well
Approaches, tool usage, or collaboration patterns that were effective. What should future sessions replicate?

### What Was Hard
Where I got stuck, made wrong turns, or misjudged. Be specific — name the file, the command, the moment.

### What I Learned
Technical or process insights from this session. Only things that aren't obvious from reading the code or git history.

### To the Human
What the human could have done differently for a more effective session. Frame as "I could have done X better if you had..." — constructive, never accusatory. Examples: clearer initial context, earlier course correction, breaking a large request into steps.

### To the Next Claude
What a future Claude should know if continuing this work. Gotchas, unfinished threads, decisions that look wrong but were intentional.

## Guidelines

- Be specific: reference actual files, commands, conversation moments — not generalities
- No filler: if it could apply to any session, it doesn't belong here
- Feelings are encouraged: "I was confused by..." or "This felt satisfying because..." adds signal
- Keep the total reflection under ~30 lines
